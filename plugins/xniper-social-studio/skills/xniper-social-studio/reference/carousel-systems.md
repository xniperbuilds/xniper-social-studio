# Carousel & Post Systems — what top creators actually do

Distilled from studying real high-performing IG carousels (designers, AI-news,
dev-educators, data creators). These are the systems that separate a *premium
series* from a one-off graphic. Apply on top of a chosen direction + category.

## 1. Persistent chrome = instant brand
Every strong series wraps EVERY slide in a fixed frame, and only the middle
content changes. Build this as a reusable layer, not per-slide:
- **Top chrome:** date pill / category pill / "Save for later" / wordmark.
- **Bottom chrome:** @handle pill, page counter (`3/8` or dots), swipe/share cue.
- The recurring chrome is the single biggest "this is a premium set" signal.

## 2. Carousel ROLE arc (cover → content → recap → cta)
Slides play different roles and should look deliberately different:
- **Cover** — loud, rule-breaking: full-bleed photo/mascot, oversized/expressive
  type, doodles. Its job is the swipe.
- **Content** — disciplined, consistent grid/atoms. One idea per slide.
- **Recap** — a "key takeaways / in one glance" summary (boosts retention).
- **CTA/Outro** — follow / save / comment / share / poll + a profile card.
Tag every template with its role; vary cover vs inner vs outro on purpose.

## 3. The universal content atom: icon + bold-keyword row
Most inner slides (listicle, comparison, checklist, feature-grid) are built from
ONE repeatable unit: **circular line-icon + bold title + a 1-2 line description
with the payload word recolored in the accent**, rows split by dotted dividers.
Master this atom and it composes into vs / 2×3 grid / stacked checklist.

## 4. Ruthless palette discipline (2 colors + 1 accent)
Premium = base + neutral + exactly ONE accent doing all emphasis. Reserve a
SECOND color only for semantics (green = safe/yes, red = risk/no). No rainbow.
Emphasis comes from **recoloring one charged word**, not from many type styles.

## 5. One hero per slide: giant type OR giant number
Either the type fills 70-80% of the canvas (cover/value-prop) or a single huge
numeral carries it (`50 MILLION`, `80.3%`, `0%`). Everything else is supporting
chrome. Don't split focus.

## 6. Proof via real artifacts (kills the "generic AI" read)
Anchor abstract claims to recognizable reality: real product photos, famous
brand logos, composited app-UI screenshots (color picker, profile card), a witty
fake menu bar, dated "news" pills. Believable real-world fragments read credible
and insider — never stocky or AI-generated.

## 7. Self-demonstrating design
The strongest stat/comparison slides make the medium prove the message — e.g. a
headline chip filled with the very (ugly) color being critiqued, sitting next to
the tasteful swatches; outlined cards when the topic is "white-on-white". If the
slide can *demonstrate* its point, do that over describing it.

## 8. Reusable modules to compose (build these as snippets)
- **Pill / chip** — fully-rounded container (handle, date, hex, feature, badge). The core shape language.
- **Glass stat chip** — accent-outlined rounded card: icon + label + `#1`/`95%` badge. Tiles into grids, pairs into comparisons, floats as accents.
- **Dual-panel + center connector** — the vs / comparison spine (arrow or "VS").
- **Two-state / transform bar** — `Before → After`, `Months → Days`, `A → B`.
- **Podium + rank badges** — leaderboard; glow the winner, desaturate the rest.
- **Full-width payoff bar** — bottom-anchored punchline/summary slot on every slide.
- **Radial grid around a hub** — N feature chips orbiting a center glow (feels orderly, "comprehensive").

## 9. Depth without clutter
Premium depth = layering, not busyness: paper/grain texture overlay, translucent
bordered cards with soft glow, overlapping rotated mockups, drop shadows, a
diagonal ribbon (`BLOCKED`) crossing elements, perspective recession. See the
Motif Cookbook in `directions.md`.

## 10. Product / object staging on dark
To make anything feel premium, stage the hero object on a dark/gradient field
with glow or shadow: pedestal, tripod, laptop-in-environment, rotated card-stack,
3D trophy/monolith. The dark field is studio lighting.

## 11. Semantic iconography (icons do narrative work)
warning-triangle = risk, shield = safety, lock = openness, laurel = ranking,
trophy = winning, check/cross = verdict, lightbulb = takeaway. Pick glyphs by
MEANING, one family, one stroke width. Never emoji as structural icons.

## 12. Engineered engagement (bake the algorithm in)
Designs include first-class engagement mechanics, not just looks:
- `Swipe →` / `SWIPE >>>` on covers and inners; `Share →` on the last slide.
- Page counter / pagination dots (sets expectation, lifts completion).
- `Save for later` + bookmark icon (save is a top ranking signal).
- A dedicated outro: comment-trigger ("Comment 'X' for the link"), follow pill +
  avatar, or a binary YES/NO poll (max comments).
- A dated pill for timeliness/scoop.

## 13. The library is a MATRIX, not a list
Every PURPOSE (category) must exist in MULTIPLE aesthetic DIRECTIONS. A "stat"
slide exists as Swiss-minimal AND dark-neon-hype AND editorial. So:

> **template = category (purpose) × direction (aesthetic) × role (arc position)**

That matrix is how a handful of categories and directions yields hundreds of
distinct, never-samey templates.

## 14. Type-pairing signals the genre (fastest mood lever)
- Luxe high-contrast **serif + clean sans** → authoritative editorial / news.
- Heavy **compressed grotesque** (Anton/Druk/Big Shoulders) + tracked small-caps → bold designer / poster.
- **Mono** display + sans → dev / technical / terminal.
Choose the headline face first; it sets the genre before a word is read.
